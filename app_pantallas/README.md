## 🎯 Objetivo de la Práctica
Implementar un sistema de navegación robusto entre pantallas en una aplicación móvil utilizando **Jetpack Compose**, comprendiendo el flujo de navegación y la estructura de navegación moderna (Single Activity Architecture).

Este proyecto simula la interfaz de una red social para practicar componentes interactivos, manejo de estados y navegación por pestañas.

---

## 📱 Características (Screens)

La aplicación cuenta con 3 pantallas principales integradas en una `BottomNavigationBar`:

1.  **Inicio (Home):** 🏠
    *   Feed de publicaciones dinámico usando `LazyColumn`.
    *   Barra superior (TopAppBar) con acciones rápidas.
    *   Interacción mediante botones de "Me gusta".
2.  **Explorar e Información:** 🔍
    *   Barra de búsqueda personalizada con diseño redondeado.
    *   Cuadrícula de contenido (Grid) optimizada.
    *   Diseño limpio para visualización de múltiples ítems.
3.  **Perfil de Usuario (Resumen):** 👤
    *   Cabecera con estadísticas (Posts, Seguidores, Seguidos).
    *   Botón interactivo de "Editar Perfil".
    *   Galería personal de publicaciones.

## 🛠️ Tecnologías Utilizadas

*   **Kotlin**: Lenguaje principal de desarrollo.
*   **Jetpack Compose**: Para la creación de la interfaz declarativa.
*   **Compose Navigation**: Gestión de rutas y grafos de navegación.
*   **Material Design 3**: Componentes de diseño moderno (Scaffold, TopAppBar, NavigationBar).
*   **Material Icons Extended**: Iconografía avanzada para mejorar la experiencia de usuario.

---

## 🧠 Conceptos Aplicados

*   **POO II**: Desarrollo orientado a objetos en entornos móviles.
*   **Flujo de Usuario**: Navegación continua y persistencia de estado entre pantallas.
*   **Componentes Interactivos**: Manejo de botones, campos de texto y listas optimizadas.

## 🚀 Cómo ejecutar el proyecto

1.  **Clonar el repositorio:**
    ```bash
    git clone https://github.com/tu-usuario/navegacion-pantallas.git
    ```
2.  **Abrir en Android Studio:**
    *   Asegúrate de tener la versión **Ladybug** o superior.
3.  **Sincronizar Gradle:**
    *   El IDE descargará automáticamente las dependencias de Compose y Navigation.
4.  **Ejecutar:**
    *   Selecciona un emulador o dispositivo físico con **API 24** (Android 7.0) o superior.
